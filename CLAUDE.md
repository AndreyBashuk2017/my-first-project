# Правила работы с проектом

## Структура файлов
- Каждая страница — один самодостаточный HTML-файл
- Никаких внешних библиотек, фреймворков и CDN-зависимостей
- CSS и JS пишем прямо внутри HTML-файла (теги `<style>` и `<script>`)
- SVG-иконки встраиваем в HTML, не подключаем внешние иконочные шрифты

## Дизайн
- Фон: тёмный (`#0a0a0a` — `#0d1117`)
- Текст: светлый (`#e2e8f0` — `#e6edf3`)
- Акцентный цвет: оранжевый (основной), дополнительно — фиолетовый и бирюзовый для градиентов
- Карточки: полупрозрачные, с тонкой рамкой (`rgba(255,255,255,0.07)`)
- Анимации: плавные, ненавязчивые (появление элементов, hover-эффекты, пульсация CTA)

## Адаптивность
- Все страницы должны корректно отображаться на телефоне
- Используем `clamp()` для шрифтов, `auto-fit` grid и `flex-wrap` для сеток
- Минимальная ширина карточек в сетке — 260–280px

## Текст и тон
- Простой, дружеский язык — без канцелярита и сложных формулировок
- Короткие предложения, конкретные выгоды
- Никаких «осуществляем», «производим», «в рамках» — пишем как говорим

## Процесс работы
- Перед созданием нового файла — сначала описать план: что будет на странице, какие блоки, какой стиль
- Дождаться подтверждения, и только потом писать код
- После создания файла — предложить запушить на GitHub
